# RAG 챗봇 시스템 명세서

## 1. 개요
본 문서는 단일 파일(`rag_chatbot.py`)로 구성된 프로덕션 수준의 RAG(Retrieval-Augmented Generation) 챗봇 시스템의 아키텍처 및 상세 설계를 정의합니다. 본 시스템은 확장성, 모듈성, 그리고 완벽한 오류 처리에 중점을 두고 설계되었습니다. 특히, 과도한 비용 발생을 방지하기 위해 완전한 로컬 무료 구동 모드를 지원합니다.

## 2. 아키텍처
시스템은 역할에 따라 논리적으로 분리된 여러 클래스로 구성되며, 주요 파이프라인은 다음과 같습니다:
`문서 로드` -> `청킹(분할)` -> `텍스트 임베딩` -> `벡터 저장소 저장` -> `사용자 쿼리 검색` -> `LLM 답변 생성`

### 2.1 주요 클래스 설계
*   **`ConfigManager`**: 시스템 설정(모델 이름, 청크 크기, 사용 모드 등)을 관리합니다.
*   **`AWSErrorHandler`**: AWS 서비스 사용 시 발생할 수 있는 오류(권한, 네트워크 등)를 상세하게 로깅하고 예외를 처리합니다. (본 프로젝트에서는 로컬 모드를 중점적으로 사용하므로, 인터페이스 형태로 구현되어 확장성을 제공합니다.)
*   **`DocumentProcessor`**: 텍스트 파일 등 원본 문서를 읽고, 적절한 크기로 분할(Chunking)합니다.
*   **`EmbeddingManager`**: 분할된 텍스트 청크를 벡터로 변환합니다. (로컬 HuggingFace 임베딩 모델 사용)
*   **`VectorStoreManager`**: 생성된 벡터를 저장하고 검색을 수행합니다. (로컬 FAISS 사용)
*   **`RAGChainManager`**: 사용자의 쿼리를 받아 검색을 수행하고, 검색된 문서를 컨텍스트로 활용하여 LLM을 통해 최종 답변을 생성합니다. (로컬 HuggingFace 파이프라인 사용)

## 3. 비용 최적화 전략 (무료 로컬 모드)
본 RAG 챗봇은 AWS(Bedrock, OpenSearch) 연동 기능을 염두에 두고 설계되었으나, 초기 개발 및 테스트 단계에서의 비용을 0으로 만들기 위해 아래와 같은 로컬 오픈소스 기술 스택을 사용합니다:
*   **임베딩**: `sentence-transformers/all-MiniLM-L6-v2`
*   **벡터 저장소**: `FAISS (cpu)`
*   **LLM (텍스트 생성)**: 로컬 자원으로 구동 가능한 가벼운 HuggingFace 모델 (예: `HuggingFaceH4/zephyr-7b-beta` 또는 더 작은 모델 사용, 메모리 한계 시 `gpt2` 등으로 폴백)

## 4. 로깅 및 에러 핸들링 전략
*   Python의 내장 `logging` 모듈을 사용하여, 각 모듈(클래스)별로 상세한 동작 상태와 에러를 기록합니다.
*   사용자에게는 친절한 메시지를 제공하고, 개발자에게는 디버깅을 위한 원시 에러 및 트레이스백을 제공합니다.
